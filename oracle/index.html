What is enterprise analytics?

Imagine a massive supermarket chain with 500 stores, 10,000 employees, and millions of transactions every day. The CEO asks: "Which stores are losing money, and why?" There's no way she can open every receipt. A company needs software that automatically collects all that data, organises it, and presents the answer as a clean chart in seconds. That software is enterprise data analytics.

Analogy

Think of enterprise analytics like Google Maps for a business. Raw data (streets, GPS pings, traffic) = the company's numbers. Google Maps = the analytics software. The route shown on screen = the answer to "why did costs go up last quarter?"

Large companies need it because humans simply cannot read millions of rows of spreadsheet data. Analytics software turns mountains of numbers into decisions.

Oracle as a company

Oracle Corporation (founded 1977) is one of the largest technology companies on Earth. Their main business: software and databases that run inside large companies — banks, hospitals, governments, retailers. Think of Oracle as the "plumbing company" of the enterprise world. Most big companies have Oracle software quietly running their HR, finance, and supply chains, even if employees never see the Oracle name.

Within Oracle's product family, the two tools you're learning sit here:

Oracle Fusion ERP / HCM / SCM
The source — the actual business systems where HR, Finance, and Supply Chain teams do their daily work. This is where all the data lives.
Fusion Data Intelligence (FDI)
The smart connector — pre-built analytics that pulls data out of Fusion apps and makes it analysis-ready instantly. Like a pre-wired electrical panel.
Oracle Analytics Cloud (OAC)
The canvas — the visualisation and reporting tool where humans actually see charts, dashboards, and insights. FDI runs on top of OAC.
Autonomous Data Warehouse (ADW)
The storage vault — a cloud database that holds all the processed data, sitting between the Fusion source and OAC's dashboards.

Key vocabulary — plain English glossary
Data Warehouse
A well-organised digital library where a company stores all its historical data, sorted and indexed for fast searching. Not where people work — where data is kept for analysis.
KPI
Key Performance Indicator — a single important number a company tracks, like "monthly revenue" or "employee turnover rate." The score on the scoreboard.
Dashboard
A single screen showing multiple charts and KPIs at once, like a car's instrument cluster. Gives a quick visual health-check of the business.
ETL
Extract, Transform, Load — the process of pulling raw data from source systems, cleaning/reshaping it, and loading it into a warehouse. Like sorting mail: collect it, sort it, file it.
Cloud
Software running on servers owned by someone else (Oracle, in this case), accessed via the internet. You don't install anything; you just log in through a browser.
SaaS
Software as a Service — you subscribe to use software like a Netflix subscription, instead of buying a CD and installing it. Oracle Fusion apps are SaaS.
Pre-built Analytics
Dashboards and reports that come ready-made, like IKEA furniture — someone else designed the data model and charts; you just configure and use them.
Semantic Model
A translation layer that explains what data means in business terms. It converts "column ID_7742" into "Employee Hire Date" so non-technical users understand what they're looking at.
Subject Area
A pre-organised collection of related data fields, like a chapter in a textbook. "Payroll Subject Area" contains every field you'd need to analyse pay data.

What problem does OAC solve?

Imagine you're a Sales Manager. Your data lives in five different places: a CRM system, an Excel file your colleague emailed, a database in IT, an old spreadsheet from 2022, and a dashboard someone built in PowerPoint. Every Monday you spend 3 hours copy-pasting these together. OAC is the tool that connects to all those sources at once, lets you drag and drop charts in minutes, and shares a live, auto-refreshing dashboard with your whole team — so Monday mornings are about decisions, not data wrangling.

How OAC compares to tools you may have heard of
vs Excel
Excel is for individual analysis on small data. OAC handles millions of rows, live data connections, and team-wide sharing. Excel breaks; OAC scales.
vs Power BI
Both do similar things. OAC integrates tightly with Oracle databases and Oracle Fusion apps. If your company runs on Oracle, OAC is the native choice — like using Apple Maps on an iPhone.
vs Tableau
Tableau has a stronger reputation for visual design flexibility. OAC has deeper Oracle database connectivity and includes AI/ML features natively. Companies in the Oracle ecosystem choose OAC.
5 things OAC can do that matter most for an intern
1. Connect to data sources
Connect to databases, files, cloud apps — like plugging different appliances into a power strip.
2. Drag-and-drop visualisations
Build charts by dragging fields onto a canvas. No code. Like assembling LEGO with data.
3. Share & publish dashboards
Publish a live dashboard to a whole team with one click. Everyone sees the same numbers.
4. AI-powered insights
OAC can automatically explain why a KPI changed, using machine learning — no data science degree needed.
5. Filters & drill-down
Click on a bar in a chart to immediately filter all other charts on the page. Interactive discovery without SQL.
Core OAC concepts with real-world analogies
Projects
A project is your working file — like a Word document that contains all your charts and analyses. You create a new project for each analysis task.
Data Sets
A data set is the raw material you bring in — like the ingredients before cooking. Could be an uploaded Excel file or a live database connection.
Visualisations
Individual charts, tables, or maps — the "dishes" you cook from your data. Each one answers one question visually.
Dashboards (Workbooks)
A collection of visualisations on one screen — like a restaurant menu board. Shows many metrics at a glance.
Data Flows
Automated pipelines that transform and prepare data before analysis — like a prep chef in a kitchen doing the chopping before cooking starts.
Semantic Models
A "translation guide" that converts database column names into business language. So "HDT_EMP_ID_7" becomes "Employee ID" in every chart, automatically.
Your very first OAC login — what you'll see

When you first log in to OAC, you'll land on the Home page. Here's what to expect step by step:

Step-by-step walkthrough

1. Home screen: A grid of "cards" showing recent projects and dashboards, plus a top navigation bar with icons for Home, Catalog, Data, and Console.

2. Create your first project: Click the blue "Create" button (top right). Select "Workbook." This opens a blank canvas.

3. Add a data set: Click "Add Data" on the left panel. You can upload a CSV file or connect to a database. For practice, upload a sample CSV.

4. Build a chart: In the data panel on the left, drag a field (e.g. "Sales") onto the canvas. OAC automatically picks a chart type. You can change it from the top toolbar.

5. Save and share: Click the save icon, give your workbook a name. Click "Share" to get a link or export to PDF.

What is FDI and what problem does it solve?

OAC is a brilliant tool — but it starts completely empty. If you're a Finance Analyst at a large company using Oracle Fusion ERP, you'd need a team of data engineers spending 6–12 months building the data pipelines, data model, and 200+ dashboard templates before OAC shows anything useful. FDI solves this: it comes with everything pre-built for Oracle Fusion. Day 1, you get 150+ dashboards, 2,000+ pre-built metrics, and connections already wired to your Fusion ERP.

Analogy

OAC alone is like buying an empty restaurant space. FDI is like buying a McDonald's franchise — the kitchen, the menu, the uniforms, and the recipes are all included. You can customise, but the core is already built and proven.

How FDI relates to Oracle Fusion applications

Oracle Fusion is a family of cloud applications: ERP (finance, accounting), HCM (HR, payroll), SCM (supply chain), and CX (customer experience). Each of these generates enormous amounts of data every day. FDI is specifically designed to understand the data structure of these Fusion apps — it knows that "WRK_ASGN_STTS_CD" in the HR database means "Employee Assignment Status." You don't have to figure that out yourself.

Key FDI concepts you must know
Subject Areas
Pre-organised buckets of related data fields for a specific business domain. "Workforce Management Subject Area" has every HR field. Like a chapter in a textbook — everything you need for one topic in one place.
Decks
A collection of KPI Cards on one screen — like a cockpit with gauges. A Finance Deck might show Revenue, Expenses, and Profit side-by-side. Executives love Decks for their daily check-in.
KPI Cards
Individual "tiles" showing one key metric with its trend (up/down arrows). Like a speedometer — one number, one direction, instantly understood. Clickable to drill into details.
Data Augmentation
The ability to bring in additional data from outside Oracle Fusion (e.g. an Excel file with budget targets) and blend it with pre-built FDI data. Like adding your own spice to an existing recipe.
Customisation
FDI lets you add your own custom metrics, rename fields, create new dashboard views, and add company-specific data — while keeping all the pre-built content intact.

Example 1 — Report requests
A manager asks: "Can you pull last quarter's headcount by department?" You open the pre-built HR dashboard in FDI, apply a date filter, and export the table to Excel in 5 minutes.
Example 2 — Dashboard prep
Before a leadership meeting, you're asked to update a KPI Deck with the latest figures. You open FDI, refresh the data pipeline, and confirm the numbers look correct before the meeting.
Example 3 — Data exploration
You're asked to investigate why overtime costs spiked last month. You open FDI's Payroll Subject Area, add an overtime filter, and use drill-down to find which department drove the spike.
Common beginner confusions about FDI

Confusion 1: "FDI and OAC are separate products." — Wrong. FDI is deployed on top of OAC. When you use FDI, you ARE using OAC under the hood. FDI just adds the Fusion-specific pre-built layer on top.

Confusion 2: "Pre-built means I can't change anything." — Wrong. Pre-built means the starting point is done for you. You can customise everything: add new fields, change visualisations, blend in new data sources.

Confusion 3: "FDI only works with Finance data." — Wrong. FDI has pre-built modules for HCM (HR), SCM (Supply Chain), ERP (Finance), and CX (Customer Experience). Each domain has its own pre-built dashboards.

Confusion 4: "FDI is the same as FAW." — Partially right. Oracle Fusion Analytics Warehouse (FAW) is the old name. It was rebranded to Fusion Data Intelligence (FDI). Same product, newer name.

FDI is built on top of OAC — what does that mean?

When Oracle engineers built FDI, they didn't start from scratch. They took OAC as the foundation and built FDI as an extra layer sitting on top. This means every FDI installation includes a full OAC instance underneath. When you look at an FDI dashboard, you are literally looking at OAC rendering the charts — FDI just added the pre-built data pipelines, the Fusion-specific semantic model, and the pre-built dashboards on top.

Analogy

Think of an iPhone and iOS. OAC is iOS — the operating system that makes everything run. FDI is a pre-installed suite of professional apps (like a fully configured business iPhone handed to a new employee). All the apps run on iOS, but someone already set them up for you.

What you can do in FDI but not plain OAC
FDI only
Pre-built Oracle Fusion pipelines. 150+ ready-made dashboards for Finance/HR/SCM. 2,000+ pre-calculated business metrics. Decks and KPI Cards UI. One-click activation of new Fusion module analytics.
OAC only (plain)
Connect to non-Oracle data sources (Google Sheets, Salesforce, CSV files). Full freedom to design custom semantic models from scratch. Advanced ML/AI augmented analytics features. Use with non-Fusion data environments.
Both (FDI includes OAC)
Drag-and-drop visualisation canvas. Data Flows for transformation. Sharing and publishing dashboards. Filters, prompts, drill-downs. Embedded analytics.

Basic data literacy — the concepts that matter most
Reading a chart
Always check: What is on the X axis? What is on the Y axis? What is the time period? What are the units? Before drawing any conclusions, answer these four questions.
Understanding KPIs
A KPI is only useful when compared to something: a target, last year's figure, or an industry benchmark. A number alone means nothing — "revenue is $5M" means nothing without "versus target of $4.5M."
What is a metric?
A metric is any number you can measure. "Headcount" is a metric. A KPI is a metric that someone has decided is strategically important. All KPIs are metrics; not all metrics are KPIs.
Correlation ≠ causation
Two numbers moving together doesn't mean one causes the other. Ice cream sales and drowning rates both rise in summer — the cause is hot weather, not each other. Always ask "what else could explain this?"
OAC navigation basics — where things live
UI map

Top navigation bar: House icon = Home (recent items). Compass = Catalog (saved workbooks). Database cylinder = Data (datasets and connections). Gear = Console (admin settings — don't touch until you understand it).

Home page: "Recent" tab shows your last opened workbooks. "Favorites" tab shows bookmarked items. Large blue "Create" button (top right) is your starting point for new work.

Inside a workbook: Left panel = data fields you can drag. Centre = canvas where charts appear. Right panel = properties for the selected chart (change chart type, colours, axis settings here). Top bar = save, share, filter, and undo buttons.

Catalog: Organised like a file system — folders and workbooks. Your company will likely have shared folders for Finance, HR, etc. Never delete from a shared folder without checking with your manager.

Exploring a pre-built FDI dashboard without breaking anything
Rule 1: Explore, don't edit
Pre-built FDI dashboards are shared with your whole company. Use View mode only. If you want to customise, always click "Save As" to create your own copy first.
Rule 2: Filters are safe
Adding or changing a filter in View mode only affects what YOU see, temporarily. It does not save or affect what other users see. Filters are your safest exploration tool.
Rule 3: Drill-down is safe
Clicking on a chart element to drill into detail doesn't change anything. Use the breadcrumb trail at the top to navigate back up.
Rule 4: When in doubt, ask
If you're unsure whether an action will affect shared content, ask a senior colleague before clicking. "Will this change affect everyone?" is always a valid question.
Introduction to filters and prompts in OAC

Filters are the most important interactive feature for day-to-day use. They let you narrow down what data the charts show, without changing the underlying data.

Dashboard Filters
Dropdowns at the top of a dashboard. Selecting "Q1 2024" in the Date filter immediately updates ALL charts on the page to show only Q1 2024 data.
Canvas Filters
Filters you add to a specific workbook canvas while building. Only affect charts on that canvas. Found in the "Filter Bar" below the toolbar.
Data Filters
Filters applied at the data set level — only certain rows are ever loaded into your analysis. Useful for large datasets where you only care about one region or department.
How to ask for help without looking lost
Specific questions win
Instead of "I don't understand this dashboard," say: "I'm trying to filter this chart by department — I found the filter bar but I'm not sure which field to use. Can you point me to the right one?"
Show what you tried
Before asking, try one or two things yourself. Then say "I tried X and got Y, but I was expecting Z — am I missing something?" This shows initiative and makes it much easier for a colleague to help.
Use Oracle docs first
For technical questions, Oracle's documentation is excellent. Search "OAC + your question" in Google — the docs.oracle.com result usually has the exact answer. Bring that link to the conversation.
